Kevin's Current Workflow
An in-depth look into how I get stuff done
Jan 26, 2013

I'm going to break this post down into sections to help you understand my workflow, how I manage time, and the apps I use. I'll be creating updated versions of this post in the future as I find new ways to improve.

Productivity and Time Management

Most people seem to think that they can manage everything in their head and that writing down your tasks seems mundane and useless. I'm here to tell you that simply isn't the case.

First off, let me stress the fact that if you think of something that needs to be done, and it takes under 5 minutes to complete the task -- DO IT IMMEDIATELY!! Read that 5 times so it really sticks.

If it takes longer than 5 minutes, always write down the task as soon as you realize something needs to be done. You can use good ol' pen and paper, but I'm often on the go, so I use a free app called Wunderlist. It's available for all platforms and auto syncs across devices.

I like to organize my tasks by each specific project I'm currently working on, then list tasks under each project name. Some people list their projects by type, i.e. Design, Front-End, etc. I find it much more useful to not do it that way. (I've tried both.) Try to be as specific as possible when creating tasks.

Design

This is my design flow - start to finish, as well as some tips and pointers.

Wireframing

I've seen a lot of designers that tend to skip the whole wireframing process. I personally think it is one of the most important steps in design, especially when doing work with a (picky) client.

If you're a designer and don't have Creative Market and Pixeden bookmarked, then go ahead and smack yourself, because you've been missing out. You'll find all kinds of wireframing templates and mock-ups to use there.

If the project you're designing for is something rather new to you (design wise), print out some browser or mobile templates and doodle on those first to get started. It's important to use a thick stroke marker during this step. You might think a pencil is better, but you want to focus on the layout -- not little details.

Once you (and your client) are happy with what's on paper, get in Photoshop or Illustrator and start going into a bit finer detail. You can use wireframing component packs or make your own.

Photoshop

I'd really like to recommend the Plugins from the awesome guys over at Source. My personal favorite is Enigma64 -- it will save you loads of time exporting png and jpg images from Photoshop. They also have a great video to help you configure Photoshop, and check out Mark Bjango's' article on color management for Photoshop.

Another useful thing to do, especially when collaborating with a team, is to define a set structure for your Groups and Layers within Photoshop. I personally like to use the following structure:

// GROUP //
. layer

I recommend naming your groups/layers the same as your id/class tags in css. That way it's easier to navigate and relate to, especially when someone other than yourself opens the PSD or stylesheet.

Image Optimization

As I stated earlier, I always use Enigma64 to export my png and jpeg images from Photoshop. I export at the highest quality possible, and then run the images through ImageOptim. If you're on Windows, you can use Shrink O'Matic.

Skala Preview

If you're on Mac and design mobile apps, Skala Preview will be your new best friend. Once connected, it will wirelessly update whatever is in your PSD to your Android or iOS device. This way as you design your app, you get a live preview of exactly what it will look like on the device. It's pure magic.

Front-End

For front end development, I use liquid templating and css preprocessing.

Mixture

Mixture is hands down the best front-end development tool on the market. I HIGHLY recommend you check it out. It has boilerplates, preprocessing, live refresh on all devices, templating, data models, server publishing, auto minification and more.

You can also connect your GitHub account to Mixture, and auto push to gh pages with the click of a button. I could go on and on about how awesome Mixture is, but just go check it out for yourself.

Liquid Templating

Another built in feature of Mixture is the Liquid Template Engine. It's similar to HTML, but adds much more functionality and power over raw HTML. I suggest you use Liquid if you use Mixture.

Bootstrap / LESS

A lot of front-end developers that use Bootstrap seem to link the complete Bootstrap stylesheet, then also link their own custom stylesheet with overrides. There is a much better way to implement Bootstrap, and it reqiures getting familiar with the LESS dynamic stylesheet language.

If you use Mixture, you can just download the bootstrap boilerplate when creating a new project, or download Bootstrap with the LESS files directly from Bootstrap's website. If you know CSS, then learning LESS will be a piece of cake, and you will be amazed by how much time it saves you. The LESS website has docs to get you started.

Using LESS with Bootstrap will make your front-end footprint much smaller. If you don't need a certain component of bootstrap, just remove the @import in bootstrap.less. Once you have all the components picked out that you'll need, you can edit them directly, along with the variables. The only downside to this, is that if Bootstrap has an update that changes a lot of things, it will be harder to implement that update.

I would argue that the smaller file size is more important, especially on high-trafficed sites. I don't see bootstrap coming out with feature changes that would force you to have to update to the newest version. If that were the case however, you would probably need to rebuild a lot of the HTML anyway, so again LESS wins.

Sublime Text

My code editor of choice is Sublime Text. I use the Monokai color theme, except I've modified the background to be a bit darker (#222). I also have the UI of the app reskinned with Soda Dark. Don't forget the Emmet plugin to write html & css a lot faster. For more tips & tricks, check out this article from Tutsplus.

Misc


BetterSnapTool

If you're on Mac, check out BetterSnapTool. It allows you to drag apps into sections you define to quickly resize windows. I have it set up to quickly get the browser on the left of the screen, and Sublime Text on the remaining space on the right.
